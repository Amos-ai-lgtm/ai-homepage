# 个人主页V1

## 1. 应用概述

**应用名称**:叶秋的个人主页

**应用描述**:一个简约清爽的个人主页网站,展示叶秋的个人信息,并提供数字分身聊天功能,供朋友、潜在合作伙伴、面试官了解和互动。

## 2. 用户与使用场景

**目标用户**:

- 朋友
- 潜在合作伙伴
- 面试官

**核心使用场景**:

- 访客浏览个人信息,快速了解叶秋的身份、兴趣和近况
- 访客通过数字分身聊天区提问,获取关于叶秋的更多信息

## 3. 页面结构与功能说明

### 3.1 页面结构

```
个人主页
├── 头部区域
│   ├── 头像
│   ├── 名字
│   └── 一句话介绍
├── 个人信息展示区
│   ├── 我现在主要在做
│   ├── 我的兴趣
│   └── 我的特点
└── 数字分身聊天区
    ├── 聊天对话框
    ├── 输入框
    └── 发送按钮
```

### 3.2 功能说明

#### 3.2.1 头部区域

**头像**:

- 展示叶秋的头像图片

**名字**:

- 展示「叶秋」

**一句话介绍**:

- 展示「一个正在学习用 AI 做产品的内容策划」

#### 3.2.2 个人信息展示区

**我现在主要在做**:

- 展示「整理自己的作品和写作方向」

**我的兴趣**:

- 展示「AI 应用、写作、动漫」

**我的特点**:

- 展示「喜欢把复杂问题讲成人话」

#### 3.2.3 数字分身聊天区

**聊天对话框**:

- 展示用户与数字分身的对话记录
- 数字分身回复基于以下知识库:
  - 职业或身份:内容策划
  - 最近在做的事:搭自己的个人主页,整理作品集
  - 擅长或关心的方向:内容表达、AI 应用、知识整理
  - 常见问题:
    - 你现在在做什么?
    - 你有哪些作品?
    - 怎么联系你?

**输入框**:

- 用户输入问题

**发送按钮**:

- 点击后将用户问题发送给数字分身,数字分身返回回复并展示在对话框中

## 4. 业务规则与逻辑

### 4.1 数字分身回复逻辑

- 数字分身根据用户提问,结合知识库内容生成回复
- 回复风格:简洁、友好、符合叶秋的表达习惯
- 对于知识库外的问题,数字分身回复「这个问题我暂时回答不了,你可以通过其他方式联系叶秋」

### 4.2 页面适配

- 页面需适配手机端,确保在移动设备上正常展示和交互

## 5. 异常与边界情况

| 场景             | 处理方式                             |
| ---------------- | ------------------------------------ |
| 用户输入为空     | 发送按钮不可点击或提示「请输入问题」 |
| 数字分身回复失败 | 提示「回复失败,请稍后重试」          |
| 头像图片加载失败 | 显示默认占位图                       |

## 6. 验收标准

1. 访客打开个人主页,看到头像、名字「叶秋」和一句话介绍「一个正在学习用 AI 做产品的内容策划」
2. 访客浏览个人信息展示区,看到「我现在主要在做」「我的兴趣」「我的特点」三项内容
3. 访客在数字分身聊天区输入问题「你现在在做什么?」
4. 访客点击发送按钮
5. 数字分身返回回复「我正在搭自己的个人主页,整理作品集」,回复展示在对话框中
